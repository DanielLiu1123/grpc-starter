# Deadline Propagation

## Problem

When a gRPC server receives a request with a deadline, the gRPC framework automatically cancels the request when the deadline expires. However, **the upstream deadline is NOT automatically propagated to any outgoing client calls** made during the same RPC.

This means that if Service A calls Service B, and the original caller set a 5-second deadline for the call to Service A, Service B may continue executing even after the original deadline has expired — wasting resources and causing cascading timeouts.

## How It Works

`grpc-starter` includes a `DeadlinePropagationClientInterceptor` that is automatically registered when the gRPC client is enabled. It works by:

1. Checking `Context.current().getDeadline()` — the gRPC framework sets this for the duration of any RPC call that has a deadline.
2. If the outgoing call options do **not** already have an explicit deadline, the interceptor applies the context deadline.
3. If the outgoing stub already has an explicit deadline set, it is **not** overridden.

## Configuration

Deadline propagation is enabled by default. To disable it:

```yaml
grpc:
  client:
    deadline-propagation:
      enabled: false
```

## Interaction with Explicit Deadlines

Explicit per-stub deadlines always take priority:

```java
// This deadline is respected and NOT overridden by propagation
stub.withDeadlineAfter(2, TimeUnit.SECONDS)
    .unaryRpc(request);

// No explicit deadline — the upstream context deadline is propagated
stub.unaryRpc(request);
```

## Example

Consider a chain: `Client → Service A → Service B`.

1. The client calls Service A with a 5-second deadline.
2. The gRPC framework sets `Context.current().getDeadline()` to 5 seconds for the duration of Service A's handler.
3. Service A calls Service B using an injected stub (no explicit deadline).
4. `DeadlinePropagationClientInterceptor` detects the context deadline and applies it to the outgoing call to Service B.
5. Service B now runs under the same deadline as Service A, and will be cancelled if the deadline expires.
