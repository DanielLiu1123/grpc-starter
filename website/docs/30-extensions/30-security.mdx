# Spring Security

The `grpc-starter-security` extension integrates Spring Security with your gRPC services, providing:

- **Bearer JWT authentication** — tokens are extracted from the `Authorization` gRPC metadata header and validated via Spring Security's `AuthenticationManager`.
- **Anonymous access** — calls without an `Authorization` header proceed as anonymous, allowing method-level security (`@PreAuthorize`) to control access.
- **SecurityContextHolder integration** — the authenticated principal is available via `SecurityContextHolder` for the duration of each RPC call.

## Quick Setup

Add the dependency:

```groovy
// build.gradle
dependencies {
    implementation("io.github.danielliu1123:grpc-starter-security")
    implementation("org.springframework.boot:spring-boot-starter-security")
    implementation("org.springframework.security:spring-security-oauth2-resource-server")
    implementation("org.springframework.security:spring-security-oauth2-jose")
}
```

Configure an `AuthenticationManager` bean. The interceptor is only registered when an `AuthenticationManager` bean is present in the application context:

```java
@Configuration
@EnableMethodSecurity
public class SecurityConfig {

    @Bean
    public JwtDecoder jwtDecoder() {
        return NimbusJwtDecoder.withJwkSetUri("https://your-auth-server/.well-known/jwks.json").build();
    }

    @Bean
    public AuthenticationManager authenticationManager(JwtDecoder jwtDecoder) {
        return new ProviderManager(new JwtAuthenticationProvider(jwtDecoder));
    }
}
```

## How It Works

For each incoming gRPC call:

1. `BearerTokenGrpcAuthenticationReader` checks the `Authorization` metadata header for a `Bearer <token>` value.
2. If a token is found, it is passed to `AuthenticationManager.authenticate()`.
   - **Success**: the authenticated `Authentication` is stored in `SecurityContextHolder`.
   - **Failure**: the call is closed immediately with `Status.UNAUTHENTICATED`.
3. If no token is present, the call proceeds anonymously (`SecurityContextHolder` remains empty).
4. After the call completes or is cancelled, `SecurityContextHolder` is always cleared.

## Using @PreAuthorize

Enable method security and annotate your gRPC service methods:

```java
@GrpcService
@EnableMethodSecurity
public class MyGrpcService extends MyServiceGrpc.MyServiceImplBase {

    @Override
    @PreAuthorize("hasRole('ADMIN')")
    public void adminOperation(Request request, StreamObserver<Response> observer) {
        // only ADMIN users can reach here
    }
}
```

## Handling AccessDeniedException

When `@PreAuthorize` denies access, Spring Security throws `AccessDeniedException`. Map it to `Status.PERMISSION_DENIED` using `@GrpcAdvice`:

```java
@GrpcAdvice
public class SecurityExceptionHandler {

    @GrpcExceptionHandler
    public Status handleAccessDenied(AccessDeniedException e) {
        return Status.PERMISSION_DENIED.withDescription(e.getMessage());
    }
}
```

## Custom Authentication Reader

To support custom token formats (e.g., API keys), implement `GrpcAuthenticationReader`:

```java
@Bean
public GrpcAuthenticationReader grpcAuthenticationReader() {
    return (call, headers) -> {
        String apiKey = headers.get(Metadata.Key.of("x-api-key", Metadata.ASCII_STRING_MARSHALLER));
        if (apiKey == null) return null;
        return new ApiKeyAuthenticationToken(apiKey);
    };
}
```

## Configuration

| Property | Default | Description |
|----------|---------|-------------|
| `grpc.security.enabled` | `true` | Enable/disable all gRPC security |
| `grpc.security.server.enabled` | `true` | Enable/disable the server interceptor |
| `grpc.security.server.order` | `50` | Interceptor order |
